3.7 Exposure Database Tools


The exposure database for the five exposure units and other elements at risk (i.e. EDT and EDM) requires tools to properly organize tabular and spatial data. Presently, there are several exposure database tools that can be used each using different platforms (i.e. software) and following (i.e. process flow). The tools that are currently use now include, but are not limited to:

       

    • Exposure Database (.xls) by HLURB-PDG
    • Dashboard and Slicers Excel Database (.xls) by UNHABITAT
    • Climex.db (qliksense) by UN RAPID Project


Exposure Database (.xls) by HLURB-PDG


The exposure database by HLURB-PDG is the default tool suggested for use by LGUs and other users. The EDT (e.g. Sample EDT for Population) presented in this manual is the main platform where data on exposure unit and hazards are organized. The main tools to facilitate the exposure database by HLURB are Microsoft Excel and GIS (particularly QGIS). Users are required to have basic to advanced knowledge in Microsoft Excel and QGIS.


Dashboard and Slicers Excel Database (.xls) by UNHABITAT


UNHABITAT built on the exposure database by HLURB and established a workflow in Microsoft Excel that makes use of Dashboard and Slicers. These functions in Excel allow users better analysis of the results of risk assessment provided that the required tabular and spatial data are available.

ClimEx.db (qliksense) by UNDP RAPID Project


The use of ClimEx.db was first suggested in the Supplemental Guidelines on Mainstreaming CCA-DRR in CLUP as a way to organize exposure data for CDRA. ClimEx.db is a tablet-based survey application designed to facilitate the gathering of geo-referenced data on population, buildings, infrastructure and economic activities in communities that are exposed to hazards and are predisposed to the impacts of climate change.


Under the RAPID Program, the ClimEx.db was enhanced to simplify its data collection and improve its data visualization component. The system has three platforms: 1) mobile application to gather data using mobile devices using Open Data Kit (ODK) Collect; 2) desktop application to visualize and manage the gathered data at the local level through the Qlik Sense Desktop; and 3) data server to host and consolidate all the gathered data. The three-step process of implementing the ClimEx.db are 1) geotagging surveys, 2) enumeration surveys and 3) data visualization and management.


Refer to the guideline Integrating the ClimEx.db to the Climate and Disaster Risk Assessment in the Formulation of the Comprehensive Land Use Plan by UNDP RAPID Project.
